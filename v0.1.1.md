## Executive Summary

v0.1.0 shipped a complete dual-deck DJ engine, groovebox, automixer, 17 skins, MIDI learn, headphone cue, and recording. The codebase is solid — ~30k lines of hand-wired TypeScript with zero audio wrappers. v0.1.1 is about **hardening**: fixing real-world bugs discovered after first public release, making MIDI controllers actually work plug-and-play, expanding settings, and building the test infrastructure that guarantees future stability.

## 0. Critical Bugs (Fix First)

### 0.1 MIDI: Akai MIDI Mix (and generic controllers) not working
- **Root Cause**: The MIDI system requires manual MIDI Learn for every single control. No auto-map preset exists. The `MidiManager.addMapping()` flow works but:
  - `isLearning` must be toggled ON from SystemHud → users don't know this
  - After clicking a UI knob, there's no visible feedback of "waiting for HW input"
  - The `learningAction` is set but instantly cleared when `addMapping` completes — no confirmation toast
  - CC values arrive normalized 0–127 → some mappings (pitch, EQ) may feel inverted or have wrong range
- **Fix Plan**:
  1. **Add Akai MIDI Mix preset** — hardcoded CC map for the most common layout (8 knobs per row × 3 rows + faders + buttons + master fader). File: `src/midi/presets/akaiMidiMix.ts`
  2. **Add generic auto-detect** — on first CC/Note message from a new device, offer "Auto Learn" vs "Load Preset"
  3. **Fix MIDI Learn UX** — add a visible banner/toast when learning, show which control is waiting, flash the target UI element
  4. **Fix CC inversion on pitch fader** — current mapping: `0 = +8%, 127 = -8%`. Akai faders need configurable polarity
  5. **Add MIDI tab in Settings** — show connected devices, current mappings table, clear/export/import/preset buttons

### 0.2 Deck B shows loaded but has no track
- We force `setDeckTrackLoaded('B', true)` at startup — this makes the deck UI visible but `isTrackLoaded = true` without an actual buffer. Attempting to play crashes silently because `transport.buffer === null`.
- **Fix**: Don't force `isTrackLoaded`. Instead, redesign the "empty deck" overlay to be subtle (show controls behind it, transparent, not the full black blur). The deck should look alive but clearly empty.

### 0.3 Settings version string hardcoded
- `SettingsModal.tsx` line 103: `Mixi v0.1.0` is hardcoded. Should read from `package.json` version.

### 0.4 GitHub repo missing files
- `tsconfig.tsbuildinfo` committed (should be in `.gitignore`)
- `fix_terminal.py` and `write_midi.py` in repo root — should be in `scripts/` or removed
- `tests/` and `vitest.config.ts` not pushed to GitHub yet (only 4 commits visible)

### 1.2 Akai MIDI Mix Default CC Map

```
Channel 1 (default):
Row 1 Knobs (CC 16-23):  Deck A: HI, MID, LOW, GAIN | Deck B: HI, MID, LOW, GAIN
Row 2 Knobs (CC 24-31):  Deck A: FLT, DLY, REV, PHA | Deck B: FLT, DLY, REV, PHA
Row 3 Knobs (CC 32-39):  (reserved / color FX)
Faders (CC 48-55):        Vol A, Vol A2, Vol B, Vol B2... → Channel faders
Master Fader (CC 56):     Crossfader position
Buttons:                  Mute = Note ON/OFF → PLAY, CUE, SYNC
```

### 1.3 Missing MIDI Actions
Currently `MidiAction` supports: CROSSFADER, DECK_GAIN, DECK_VOL, DECK_EQ_*, DECK_FILTER, DECK_PLAY, DECK_SYNC, DECK_CUE, DECK_PITCH, GROOVEBOX_PAD.

**Missing actions to add**:
- `MASTER_VOL` — master volume CC
- `HEADPHONE_MIX` — CUE/MASTER mix knob
- `HEADPHONE_LEVEL` — headphone volume
- `DECK_HOT_CUE` — trigger hot cue by index
- `DECK_LOOP` — set auto-loop beats
- `FX_KNOB` — per-deck FX amount
- `FX_TOGGLE` — per-deck FX on/off

## 7. Code Quality & Debt

### 7.1 TypeScript Strictness
- `midiAction?: any` used in 6+ components — should be typed as `MidiAction | undefined`
- `(window as any).__MIXIMIDILEARN__` — global bus should be a proper typed EventEmitter or Zustand slice
- Several `as unknown as` casts in SystemHud and SettingsModal

### 7.2 CSS Audit
- Mixed Tailwind utility classes and CSS custom properties
- Some components use hardcoded `rgba()` values instead of skin variables
- `index.css` is 26KB — could benefit from splitting into `base.css`, `components.css`, `skins.css`

### 7.3 Build Artifacts
- `tsconfig.tsbuildinfo` should be in `.gitignore`
- `fix_terminal.py`, `write_midi.py` in repo root — move to `scripts/` or remove
- `api/build/`, `api/dist/` (PyInstaller output) — add to `.gitignore`
- `release/` directory — add to `.gitignore`
